**This is the first public release of `snipemgr`.** It ships with the full feature set built across Phases 0–4 plus the interactive install picker added in this version.

---

## Interactive install picker

`snipemgr install` no longer requires a name argument. Run it without one in an interactive terminal and it fetches the live registry, then presents a scrollable picker of every available integration — no need to run `snipemgr list` first or remember the exact repo name.

```
$ snipemgr install

  Select an integration to install
  > 1Password — Sync 1Password Business members → Snipe-IT
    GitHub — Sync GitHub org members → Snipe-IT
    Google Workspace — Sync Google Workspace licenses → Snipe-IT
    Okta — Sync Okta members → Snipe-IT
    Slack — Sync Slack billable members → Snipe-IT
    My integration is not listed...
```

Selecting **"My integration is not listed..."** prints step-by-step instructions for adding a third-party owner to `registry.sources` in `snipemgr.yaml` so their integrations appear in future runs.

`snipemgr install <name>` continues to work exactly as before. `--no-interactive` and piped usage still require an explicit name argument.

---

## `snipemgr upgrade`

New command to keep installed integrations current.

```bash
snipemgr upgrade          # prompt before each outdated integration
snipemgr upgrade --all    # upgrade everything without prompting
```

Upgrade replaces the binary only — `settings.yaml` is never touched. After downloading, `upgrade` compares the new manifest's config fields against the existing settings file and warns about any newly-added fields so you can run `snipemgr config <name>` to fill them in.

## Update indicators in `list` and `status`

Both commands now show a `↑ update` indicator when a newer version is available in the registry. `status` also gains a **VERSION** column.

## Timezone-aware Cloud Scheduler

The install wizard now prompts for a timezone when scheduling a Cloud Run Job. Defaults to UTC; configurable via `gcp.scheduler_timezone` in `snipemgr.yaml` for non-interactive installs.

## Cross-platform release workflow

`.github/workflows/release.yml` builds and publishes binaries for macOS (arm64/amd64), Linux (amd64/arm64), and Windows (amd64) on every `v*` tag push. A `checksums.txt` with SHA256 hashes of all binaries is included in every release for download verification.

---

## Installation

Pre-built binaries are attached to this release.

```bash
# macOS (Apple Silicon)
curl -L https://github.com/jackvaughanjr/2snipe-manager/releases/download/v1.1.0/snipemgr-darwin-arm64 -o snipemgr && chmod +x snipemgr

# macOS (Intel)
curl -L https://github.com/jackvaughanjr/2snipe-manager/releases/download/v1.1.0/snipemgr-darwin-amd64 -o snipemgr && chmod +x snipemgr

# Linux (amd64)
curl -L https://github.com/jackvaughanjr/2snipe-manager/releases/download/v1.1.0/snipemgr-linux-amd64 -o snipemgr && chmod +x snipemgr

# Linux (arm64)
curl -L https://github.com/jackvaughanjr/2snipe-manager/releases/download/v1.1.0/snipemgr-linux-arm64 -o snipemgr && chmod +x snipemgr
```

Or build from source:

```bash
git clone https://github.com/jackvaughanjr/2snipe-manager
cd 2snipe-manager
go build -o snipemgr .
```
